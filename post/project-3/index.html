Mask-off Detection (Smart City)

During COVID-19 pandemic, Singapore/Thailand/Vietnam has made wearing face mask mandatory in public.

A lot of surveillance systems are integrating computer vision in order to monitor whether people are wearing masks or not.

Challenge:

In this challenge you will build a system that is able to distinguish between people wearing a mask vs people not wearing a mask:

• Train the inference model from a dataset of faces with a mask and without a mask

• Optimize the model for the hardware and deploy the inference model to an edge device.

• For predictions with low confidence, the images are to be sent from the edge to cloud for retraining purposes.

Our Vision:

Our project aims to lighten the load on the shoulders of safe-distancing ambassadors through image recognition.

The Covid-19 pandemic has been a long and enduring pandemic, in order to ensure the population does not spread the virus while in public, mask mandates have been put in place.

However, not everyone is compliant. By using image recognition to identify offending patrons, we can reduce the number of safe-distancing ambassadors required to patrol the mall and get them to reach the incorrectly masked patrons as needed.

Functinal Requirements:

Business Solution:

By implementing our product, we are able to reduce manpower and improve on the efficiency of the on-site officers, in this case, the Safe-Distancing Ambassadors that are assigned to a location.

Cameras would be placed around commonly patronised areas to check for crowd density and unmasked individuals so that the Safe-Distancing Ambassadors will be directed to a certain area in the location when the crowd density of that area hits a certain threshold.

Based on this feature implemented in our prototype, the Safe-Distancing Ambassadors should mainly depend on the system to redirect them to that specified area to monitor the crowd and ensure that Safe-Distancing Measures are abided by the public.

With this solution in mind, we can allow businesses to concentrate on what is important, whether it be organising sales, promoting goods or managing rented stores. The system will handle some of the new responsibilities they have had to take on since the pandemic, hence helping them return closer to a state or normal, where they need not worry about the mechanics of handling pandemic crowds.

Super User Guide

Creating/Editing Users

Users may be created or edited under the Users tab.

We may choose ‘Add new user’ to add a new user, or ‘Edit’ to edit a user. ‘Delete’ opens a prompt to confirm your choice before executing the action. Here we can specify a User ID, Email, UserName and Password. The User ID has to be unique and is used to identify users, Email allows us to retain contact information with the particular user and Password is for the user to log in with. UserNames are required but are not used to identify users.

Regular User Guide

Regular Users have access to multiple tabs, ‘Cameras’, ‘Contacts’, ‘Analytics’ and ‘File Directory’.

View Cameras

The ‘Cameras’ tab allows users to create and view cameras from the system. Connected Camaras allows users to use the system for mask detection. Administrators would be able to view the cameras that are located in the target location. Camera records will be shown to the user in a tabular format.

Creation of Raw Stream

By clicking on the “Click here for creation of stream” link, the administrators will be redirected to another web page where they would need to configure stream details and AWS credentials before creating and starting the raw stream.

Administrators would be able to see that the stream is being created through the “Logs” section.

Starting Raw Stream

If the stream has just been created or the camera is not turned on, the administrator should click on “Start Master” to start the stream.

If the camera is already turned on, the administrator can just click on the “Start Viewer” button to view the stream currently running.

Starting Image Recognition Stream

Users are to click on the “Opens webcam” button to start the Image Recognition Stream. This will run the code to open their webcam as well as the Image Recognition model to start detecting their face

Start Image Recognition Training

Users are to click on the “Starts training” button to start the Image Recognition code to start training a new model.

Using Analytics Tab

The ‘Analytics’ tab allows users to view statistics and graphical information from uploaded CSV files from the cameras. Users are allowed to upload a CSV file to allow them to view monthly statistics.

Creating/Editing Contracts

The ‘Contacts’ tab allows users to create, edit and delete contacts from the system. Contacts are safe-distancing ambassadors that have been assigned to the system, allowing the system to alert them to improperly masked individuals. It takes in minimal information, only what is required to contact the safe-distancing ambassadors.

We may create a new contact by clicking the Create New Contact button. Here we may enter information to associate the contact with their assigned location and organisation that manages the location, as well as crucial information regarding them. Information from the database regarding locations, organisations and notification ids are pulled and neatly displayed in drop-down menus. It is not necessary for the user to know the ID of each location and organisation.

The Edit Contacts menu is similar, each box is filled automatically based on which contact is retrieved. However, a delete button is present, allowing us to remove contacts from the database if necessary.

Navigation of the ‘Contacts’ tab is simple, each location is a link, in blue text, clicking it retrieves a table of contacts assigned to the site. A search bar offers support for filtering through contacts by name.

Creating/Editing Organisation

Only the Regular User will be able to create Organisations and edit Organisation’s details under the Organisation tab:

Regular User can click on “Add New Organisation” to add a new Organisation into the system. Example of a Organisation is “Frasers Centrepoint” Editing of the Organisation’s details can be conducted only by the Regular User. “Organisation ID” field is a unique field that cannot be edited.

Creating/Editing Location

Only the Regular User will be able to create Locations and edit Location’s details under the Locations tab:

Regular User can click on “Add New Location” to add a new Location into the system. Example of a Location is “CausewayPoint”. Editing of the Location’s details can be conducted only by the Regular User. “Location ID” field is a unique field that cannot be edited.

Public User Guide

Public Users may access the website without login to view information about the development team and their contacts

Installation Guide

  1. Make sure you have Anaconda installed (https://repo.anaconda.com/archive/)

  2. Look for Anaconda3-2019.07-Windows-x86_64

  3. Open Anaconda Prompt and Use the Command (conda create –prefix “FULLPATH_TO_WORKING_DIR\venv” python=3.7) FULLPATH_TO_WORKING_DIR refers to wherever your repo is located

  4. Open Command Prompt and Use (python -m pip install -r venv-dependencies.txt)

  5. Open Anaconda Prompt and Use (conda install dlib)

  6. Conda Version should not be lower than 4.7.10, if it is update it using (conda update conda)
