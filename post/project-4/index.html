C300 Drowning Detection

Due to a collaborative study found by Singapore General Hospital and Singapore Civil Defence Force, there are seven out of ten drowning cases in Singapore and four out of ten of these cases are children younger than ten years old.

Our project aims to create a solution to assist in the detection of early signs of drowning.

This is accomplished in the aim of reducing the frequency of drownings in Singapore swimming pools. This implementation is carried out by combining several technologies.

Deep learning technology is used to create an object detection model, run the model on an inference stream, integrate the inference stream into a native application, and allow the Internet of Things (IoT) device to sound an alarm to inform people around the swimming pool when drowning is detected.

The Drowning Detector is useful for both supervised and unsupervised swimming pools. When a person exhibits signs of drowning, an audible alarm is raised. The first responders’ attention is drawn to the person in distress by a red box on their screen.

For supervised pools, lifeguards can use phones or other devices to monitor multiple cameras. Incidents will immediately draw their attention enabling quick assessment of the situation using their own (human) experience. If necessary they can take quick action.

For unsupervised pools, alarms and display information can be sent to security, guard houses or other designated first responders.

Project Objective

Our project is trying to reduce the frequency of drowning cases in swimming pools by having efficient responses to the drowning events. The purpose of this project is to detect early signs of drowning and alert the first-responders to take the necessary actions.

User Guide

Using the Administrator account described above, the user would be redirected to an Admin Homepage where they could access the various pages made available to Administrators.

Home Page

The user; or administrator; would then be redirected to the administrator’s home page after successfully logging into the application.

The administrator would be able to do the following on this screen: Navigate to the Detection System To view the available cameras installed Navigate to the Administrator Page To manage users of the application. Open a drawer to see the various shortcuts made available to him/her.

By tapping on “View More,” the administrator can access additional pages.

Side Menu

On the top left of the Home Page for the Administrator, there are 3 lines that are tappable, which opens a drawer, as presented on the left.

After opening the drawer, there are multiple features on the drawer itself: Their Profile (Username and Email) Shortcuts to:

  • Home Page
  • Detection Systems
  • Google Maps
  • Profile Page
  • Logout Button

These shortcuts allows the administrator to access the pages quicker and more efficiently.

Feature Page

The administrator would be able to access the following page after tapping on “View More” in the homepage:

  • Detection System
  • Pool Locations
  • Profile Page
  • Admin Panel

Profile Page

On this screen, the administrator can view their previously registered information such as their Username, Email, Gender, and Phone Number.

The administrator would be able to upload a profile picture that would be stored in Firebase.

Admin Page

The administrator would be able to manage the users by going to the Administrator Page from this page.

The administrator would be able to add, edit, and delete users from the application, and the Firebase database would be updated as a result.

Create User

The Administrator would have access to this page in order to create a user for a Lifeguard.

The details that needs to be implemented into the fields are:

  • Name,
  • Email Address
  • Password (Decided by Administrator)
  • Gender
  • Phone Number

Once created, the user will be able to log in to the device with the ‘Lifeguard’/‘Guest’ permissions.

Edit User

The Administrator would be able to edit an existing user’s rights on this page to allow them to gain administrator/lifeguard rights, depending on their needs.

Edit User rights

After entering an existing user’s email address, click the “Get User” button to obtain specific user information such as their:

  • Email Address
  • Username
  • Phone Number
  • Current User Rights

If the Administrator decides to grant administrator privileges to a specific user, he or she can do so by tapping on “Edit User Rights” and being redirected to the next screen.

Admin Panel

On this Admin Panel Screen, the administrator would be able to change the user rights by tapping on the text field to change their user rights.

This would also be updated accordingly with the database whereby the values would be updated in real-time.

Detection System

On this Detection System Screen, the administrator will be able to access the real-time detection stream by tapping on the “Available Camera”.

LiveStream Page

Installation Guide

  1. Do make a request to the email stated in Appendix A to access the GitHub respository (located in the Appendices section).
  2. Ensure that you have Visual Studio Code (VSC) installed.
  3. Install the “Flutter” and “Dart” extensions on Visual Studio Code
  4. Do a “git clone https://github.com/ngicolee/C300_DrowningDetection.git” to a folder that you would like the project to be stored in.
  5. Open Visual Studio Code, Click on “Open Folder…”, and select the cloned project repository (stored in the previous step).
  6. Install any packages (if prompted).
  7. On your phone, go into your phone Settings and turn on “USB debugging”
  8. Connect a USB cable from your computer to your phone.
  9. On Visual Studio Code, Ctrl+Shift+P to open a Command Palette, type in “Flutter: Select Device” and select your phone (device)
  10. Drill down the project folder “C300_DrowningDetection > lib > widgets > main.dart”.
  11. On the top ribbon, click on the “Run” tab and select the “Start Debugging” with the main.dart file opened.
  12. The mobile application will be installed on your phone.
